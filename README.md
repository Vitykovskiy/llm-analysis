# Программная документация

(ЕСПД, ГОСТ Р 19)

## 1. Общие сведения

Программный продукт **LLM Analysis** представляет собой веб-ориентированную информационную систему, предназначенную для автоматизации процессов сбора, уточнения и формализации требований к информационным системам с использованием LLM-агента, исполняющего роль системного аналитика.

Разработка выполнена в рамках выпускной квалификационной работы и ориентирована на применение в условиях малого бизнеса, где отсутствует штатная роль системного аналитика.

---

## 2. Описание применения программы

### Назначение программы

Программа предназначена для поддержки аналитической деятельности на предпроектном этапе разработки информационных систем и обеспечивает:

- диалоговый сбор неструктурированных требований, сформулированных на естественном языке;
- последовательное уточнение целей, границ автоматизации и ролей пользователей;
- формализацию требований в виде аналитических артефактов;
- структурирование аналитической деятельности в виде иерархии задач;
- фиксацию и накопление знаний, полученных в ходе анализа.

Программа не предназначена для управления проектной деятельностью и не заменяет системы управления разработкой, а используется как аналитический инструмент.

### Область применения

Система ориентирована на:

- индивидуальных предпринимателей и малый бизнес;
- разработчиков, выполняющих анализ требований самостоятельно;
- использование в учебных и исследовательских целях при моделировании процессов формирования требований.

### Среда функционирования

- серверная часть: платформа Node.js;
- клиентская часть: веб-браузер (современные версии Chrome, Firefox, Edge);
- хранилище структурированных данных: реляционная СУБД SQLite;
- хранилище текстовых фрагментов: векторная база данных Chroma (опционально);
- взаимодействие с LLM осуществляется через внешний API-провайдер, совместимый с LangChain.

## 3. Текст программы

### Используемые языки и технологии

- TypeScript;
- фреймворк NestJS (серверная часть);
- Vue 3 и Vuetify (клиентская часть);
- LangChain (интеграция с LLM);
- SQLite и Chroma (хранение данных).

### Основные программные модули

#### Серверная часть

- **Модуль диалогового анализа требований**  
  Обеспечивает приём пользовательских описаний, взаимодействие с LLM-агентом и сохранение результатов диалога.

- **Модуль управления аналитическими задачами**  
  Реализует декомпозицию аналитической деятельности (эпики, задачи, задачи уточнения), хранение статусов и связей между задачами.

- **Модуль интеграции с LLM**  
  Формирует запросы к языковой модели, управляет последовательностью аналитических шагов и использованием вспомогательных инструментов.

- **Модуль хранения данных**  
  Обеспечивает сохранение структурированных данных в реляционной базе и текстовых фрагментов во векторном хранилище.

#### Клиентская часть

- **Интерфейс диалогового взаимодействия**  
  Используется для ввода исходных требований и просмотра истории аналитического диалога.

- **Интерфейс визуализации структуры анализа**  
  Отображает иерархию аналитических задач и их текущее состояние.

---

## 4. Содержание программы

### Структура программного продукта

- серверное приложение (REST API, логика агента, хранилища данных);
- клиентское веб-приложение (SPA);
- конфигурационные файлы среды выполнения.

### Хранимые данные

- структурированные данные:
  - аналитические задачи;
  - статусы выполнения;
  - связи между задачами;
- неструктурированные данные:
  - текстовые фрагменты требований;
  - ответы и уточнения пользователя;
  - результаты промежуточного анализа.

---

## 5. Web-представительство

### Архитектура пользовательского интерфейса

Пользовательский интерфейс реализован в виде одностраничного веб-приложения. Взаимодействие с серверной частью осуществляется по протоколу HTTP с использованием JSON-обмена.

### Основные экраны

- **Экран диалога анализа требований**  
  Предназначен для ввода описаний задачи автоматизации, просмотра ответов LLM-агента и фиксации выявленных фактов.

- **Экран структуры аналитической деятельности**  
  Отображает декомпозицию анализа в виде задач, эпиков и уточнений, а также их взаимосвязи.

---

## 6. Руководство программиста

### Установка и настройка

Для функционирования системы требуется установка среды выполнения Node.js и настройка переменных окружения, включая ключ доступа к LLM-провайдеру. Использование векторного хранилища является опциональным.

### Расширение системы

Система допускает расширение:

- добавлением новых аналитических инструментов LLM-агента;
- модификацией структуры аналитических задач;
- подключением альтернативных хранилищ данных.

---

## 7. Руководство оператора

### Подготовка к работе

Перед началом работы необходимо:

- убедиться в корректности параметров окружения;
- обеспечить доступ к LLM-API;
- проверить доступность хранилищ данных.

### Работа с системой

Оператор (пользователь системы):

1. вводит исходное описание задачи автоматизации;
2. отвечает на уточняющие вопросы агента;
3. просматривает формируемые аналитические задачи и результаты;
4. использует полученные формализованные требования на этапах проектирования.

### Резервное копирование

Резервное копирование осуществляется путём сохранения файлов реляционной базы данных при остановленной серверной части.

---

## 8. Ограничения и допущения

- качество результатов зависит от полноты и корректности вводимой пользователем информации;
- система не заменяет экспертизу системного аналитика при сложных и масштабных проектах;
- результаты анализа носят рекомендательный характер и требуют экспертной проверки.

---

## 9. Заключение

Программный продукт **LLM Analysis** обеспечивает инструментальную поддержку аналитической деятельности на ранних этапах жизненного цикла информационных систем. Использование LLM-агента позволяет повысить полноту и согласованность требований в условиях ограниченных ресурсов малого бизнеса и может рассматриваться как практическая реализация задач, поставленных в выпускной квалификационной работе.
